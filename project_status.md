# ✈️ Travel Expense Manager 프로젝트 현황

## 1. 프로젝트 개요
- **목적**: 여행별 지출 내역 관리 및 통계 시각화 서비스
- **기술 스택**: Python 3.12, Django, SQLite, Chart.js

## 2. 현재 디렉토리 구조 (주요 경로)
- `apps/`: 서비스 핵심 비즈니스 로직 (accounts, dashboard, main, transactions, trips)
- `config/`: Django 설정 및 메인 URL 설정
- `core/`: 공통 유틸리티 및 커스텀 검증기(Validators)
- `templates/`: HTML 프론트엔드 파일
- `static/`: CSS 및 프론트엔드 자산

## 3. 구현된 주요 기능
### ✅ 사용자 관리 (Accounts)
- 로그인 / 회원가입 기능

### ✅ 대시보드 (Dashboard)
- 사용자별 맞춤형 데이터 요약 조회
- 개인별 여행 현황 및 지출 통계 대시보드 제공

### ✅ 여행 관리 (Trips)
- 여행지 등록, 수정, 삭제 및 목록 조회
- 여행별 기간 및 메모 관리

### ✅ 지출 관리 (Transactions)
- 지출/입금 내역 등록 및 카테고리 분류
- 여행 정보 연동을 통한 지출 기록
- 영수증 파일 업로드 기능

### ✅ 메인 대시보드 (Main)
- 최근 여행지 목록 표시
- 서비스 전체 통계 (총 사용자, 총 여행 수, 누적 지출)
- 국가별 지출 TOP 3 시각화 (Chart.js 막대 그래프 적용)

## 4. 진행 예정 사항
- 1) 대시보드 (이번달입금, 이번달출금, 순액 )-> 최근6개월 입금, 출금, 잔액 ?
- 2) 영수증 첨부하면 자동으로 거래내역 등록(첨부파일 파싱) 
- 3) 사용자별 지출 리포트 내보내기 (CSV/PDF)
- 4) 여러사용자가 하나의 여행과 거래내역을 같이 공유할 수 있는 기능

## 5. 기타 메모

- 국가별 통계는 `transaction_type='expense'` 데이터만 필터링하여 계산됨
